# Talash AI — Production Agent Pipeline

This plan outlines the architecture and implementation steps for INSAAF OS, Pakistan's AI legal guidance system, strictly following the provided `prompt.txt`.

## User Review Required

> [!IMPORTANT]
> The plan uses `google-adk` for all agents as required. Since `google-adk` agents typically rely on a static initialization, I will implement the API key rotation by dynamically updating `os.environ["GEMINI_API_KEY"]` and wrapping agent executions in a retry loop inside `gemini_client.py`. Please let me know if you prefer a different method for key rotation within the ADK.

> [!WARNING]
> PDF generation via `reportlab` will include basic formatting to ensure robustness without complex layouts that might fail under partial agent outputs.

## Proposed Changes

---

### Core Infrastructure

#### [NEW] `gemini_client.py`
Implement `API key rotation` and `FAULT TOLERANCE`.
- Load `GEMINI_API_KEY_1`, `GEMINI_API_KEY_2`, `GEMINI_API_KEY_3`.
- A wrapper `run_agent_with_retry(agent, prompt, context)` that catches 429 exceptions, automatically sets the next API key in the environment, and retries.
- Wait 60 seconds if all keys are exhausted.

#### [MODIFY] `requirements.txt`
- Populate with exactly the requested dependencies: `google-adk`, `google-genai`, `pinecone-client`, `python-dotenv`, `fastapi`, `uvicorn`, `reportlab`, `speechrecognition`, `pyaudio`, `pypdf`.

---

### Tools

#### [MODIFY] `tools/search_legal_docs.py`
- Initialize `Pinecone` using env variables.
- Use `google-genai` `embedding-001` for embeddings.
- Filter results > 0.3 score, top_k = 10, return formatted string.
- Catch all errors and return an empty string fallback.

#### [MODIFY] `tools/voice_input.py`
- Use `speechrecognition` with `language="ur-PK"`.
- Apply ambient noise adjustment.
- Return `None` on failure to fallback to text input.

---

### Agent Pipeline (Google ADK)

For all agents, I will define them using `google.adk.agents.llm_agent.Agent` (or equivalent syntax) with their specific System Prompts and Output Schemas defined as Pydantic models for structured JSON output.

#### [MODIFY] `agents/case_listener.py` (Agent 1)
- Input: Text/Voice transcript. Output: JSON (language, facts, emotional tone, people, assets).

#### [MODIFY] `agents/case_classifier.py` (Agent 2)
- Input: Agent 1 output. Output: JSON (category, confidence, jurisdiction, urgency).

#### [MODIFY] `agents/questioning_agent.py` (Agent 3)
- Generates up to 4 yes/no questions based on category to fill gaps.

#### [MODIFY] `agents/rights_analyzer.py` (Agent 4)
- Has access to the `search_legal_docs` tool.
- Finds relevant laws, outputs up to 5 rights, and considers emotional tone.

#### [MODIFY] `agents/document_checker.py` (Agent 5)
- Computes Readiness Score based on missing/found documents for the specific case type.

#### [MODIFY] `agents/action_planner.py` (Agent 6)
- Has access to `search_legal_docs`.
- Creates a 6-step actionable plan with costs, times, and free legal aid options.

#### [MODIFY] `agents/misguide_detector.py` (Agent 7)
- Flags scams/fraud risks and provides protective questions. NEVER fails completely.

#### [MODIFY] `agents/pdf_formatter.py` (Agent 8)
- Generates 3 PDFs (Summary, Legal Draft, Aid Request) using `reportlab`.

---

### Orchestration and Servers

#### [MODIFY] `pipeline.py`
- Orchestrates Agent 1 through 8.
- Pauses after Agent 3 to collect user answers (passed in or handled via callback/generator).
- Uses fallback objects if any agent fails, ensuring completion.

#### [MODIFY] `main.py`
- CLI Entry point. Displays the splash screen.
- Asks for text or voice input.
- Displays progress for each agent execution.

#### [NEW] `api.py`
- FastAPI server with `/analyze` endpoint (supports `StreamingResponse` for progress) and `/health`.
