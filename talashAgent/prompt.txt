API KEY ROTATION — CRITICAL
════════════════════════════════════════
Implement this in a shared gemini_client.py:

- Load all 3 API keys from env
- current_key_index = 0
- If any Gemini call gets 429 error:
  → rotate to next key automatically
  → retry the same call
  → if all 3 keys exhausted: wait 60 sec then retry
- This applies to ALL agents and ALL Gemini calls

════════════════════════════════════════
FAULT TOLERANCE — PRODUCTION CRITICAL
════════════════════════════════════════
Every agent must:
1. Have try/except around all operations
2. On failure return a fallback object:
   {
     "agent": "AgentName",
     "status": "failed",
     "fallback_used": true,
     "partial_output": {},
     "reason": "error message",
     "instruction_to_next_agent": 
       "use available data and proceed"
   }
3. Next agent MUST accept fallback and continue
4. Pipeline must reach Agent 8 regardless
5. User must NEVER see raw error messages

════════════════════════════════════════
EMOTION HANDLING — ALL AGENTS
════════════════════════════════════════
Agent 1 detects emotional_tone:
  distressed | calm | confused | urgent

All subsequent agents receive emotional_tone
and adjust their response style:

- distressed → start with empathy, 
  simple words, reassuring tone
- urgent → highlight time-sensitive 
  actions first, bold warnings
- confused → explain step by step, 
  avoid legal jargon completely
- calm → normal professional tone
